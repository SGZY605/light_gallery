# Dashboard 外壳控制设计

## 目标

优化登录后的 dashboard 体验：导航只高亮当前页面，侧栏底部提供深浅色切换和宽窄布局切换，默认进入宽模式，并略微缩窄侧栏，让主窗口更沉浸。

## 交互

左侧导航的“首页”只在路径等于 `/dashboard` 时高亮；其他导航项在路径等于自身或进入其子路径时高亮。

侧栏底部新增两个图标按钮，使用 lucide 图标并沿用导航项的低对比、细线条、柔和 hover 样式。第一个按钮切换深色/浅色主题，第二个按钮切换宽模式/窄模式。两个偏好都写入 `localStorage`，刷新后保持用户选择。

## 布局

登录后默认宽模式，dashboard 主外壳使用全浏览器宽度。窄模式沿用当前 `max-w-[1500px] mx-auto` 的内容宽度。桌面侧栏宽度从 `xl:w-48` 收窄到 `xl:w-44`，移动端仍保持顶部横向区域。

## 视觉

全局颜色改为 CSS 变量驱动。深色模式提高默认文字和背景对比，浅色模式使用温和的浅色背景和深灰文字。为了兼容现有页面里大量 `text-white/*`、`border-white/*` 和 `bg-black/*` 工具类，先在全局样式中为 dashboard 根节点添加主题覆盖规则，避免大规模重写业务页面。

## 测试

新增纯函数单元测试覆盖导航高亮、主题偏好和布局偏好。实现后运行单元测试、lint，并通过浏览器访问 dashboard 页面，检查深浅色和宽窄模式切换是否生效。
