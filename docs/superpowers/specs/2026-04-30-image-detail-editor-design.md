# 图片详情页重构设计

## 目标

重构图片详情页，使其成为一个稳定、专业、可编辑的全屏看图界面。该界面需要同时满足以下目标：

- 从图片缩略图进入独立的全屏详情页，而不是悬浮窗。
- 从缩略图到详情页主图提供平滑的放大过渡动画。
- 使用当前图片生成高斯模糊磨砂玻璃背景，强化沉浸感。
- 支持可靠的滚轮缩放、按钮缩放、拖动平移和重置缩放。
- 右侧以专业看图工具的方式展示摄影友好的元数据。
- 标签和地理位置编辑改为统一草稿态，并通过同一个保存按钮一次性提交。
- 在存在未保存更改时，关闭详情页前先弹出确认。

## 当前问题

现有实现已经引入了独立路由、侧栏、地图和部分保存接口，但整体仍处于半成品状态，主要问题如下：

- 图片开场动画与缩放拖动共用了同一层 `transform`，导致缩放和平移状态不稳定，实际交互不可用。
- 右侧信息栏层级、留白、对比度和滚动边界不足，文字不明显，区块之间缺少稳定结构。
- 标签编辑采用即时保存，位置编辑采用单独保存，两者和“统一保存”的需求冲突。
- 地图使用默认 marker，摄影位置语义不明确，经纬度输入、地图点选和拖动之间没有形成统一草稿流。
- 关闭操作没有统一经过未保存状态守卫。

## 核心决策

- 详情页继续使用独立路由 `/dashboard/library/[id]`，不回退为弹窗或抽屉。
- 查看器交互与入场动画彻底解耦：动画层只负责入场，交互层只负责缩放和平移。
- 右侧使用固定宽度、独立滚动、底部操作区固定的专业信息栏。
- 标签和位置编辑统一进入草稿态，只有点击“保存更改”时才一起提交。
- 保存接口改为聚合式更新，在单个事务中同时更新标签和位置。
- EXIF 展示采用“双层结构”：上半部分展示整理后的专业字段，下半部分提供折叠的“更多元数据”区，尽量完整呈现可读的原始摄影字段。

## 页面结构

详情页分为左右两大区域：

- 左侧为图片查看器，占据主要屏幕空间。
- 右侧为图片信息与编辑侧栏，在桌面端固定宽度显示，在小屏设备上纵向堆叠。

左侧查看器包含：

- 顶部工具条：关闭、缩小、当前缩放百分比、放大、重置。
- 中央图片舞台：承载开场动画、图片缩放与拖动。
- 底部文件名栏：显示当前图片名称。

右侧信息栏包含：

- 文件信息
- 拍摄信息
- 相机与镜头信息
- 曝光参数
- GPS 与位置说明
- 标签编辑区
- 地图与位置编辑区
- 更多元数据折叠区
- 固定底部操作区

## 缩略图到详情页的过渡

列表页点击缩略图时，前端记录当前缩略图在视口中的位置和尺寸。详情页加载后：

1. 使用记录到的缩略图矩形作为入场动画初始状态。
2. 主图从缩略图位置和尺寸平滑过渡到详情页目标位置和尺寸。
3. 入场动画完成后，动画层退出控制，交互层接管缩放和平移。

如果浏览器无法访问 `sessionStorage` 或没有记录到缩略图矩形，则直接使用淡入和轻微缩放作为降级动画。

## 图片查看器交互

### 视觉层次

- 最底层使用当前图片的预览图生成全屏模糊背景。
- 背景在模糊基础上叠加较深的遮罩，保证前景图和侧栏文字清晰可读。
- 主图使用独立的前景层，不和背景层共享滤镜或变换。

### 变换模型

查看器采用双层变换结构：

- 外层动画容器：只处理进入详情页时的过渡动画。
- 内层交互容器：只处理 `translate` 和 `scale`。

这样可以避免动画库的 `transform` 与手工缩放拖动的 `transform` 互相覆盖。

### 缩放规则

- 默认缩放为适屏显示，界面文案显示为 `100%`。
- 支持滚轮缩放，缩放中心跟随当前鼠标位置。
- 支持工具条按钮放大、缩小和重置。
- 缩放范围限制在合理区间，建议为 `25%` 到 `600%`。

### 拖动规则

- 仅当当前图片超出适屏范围时启用拖动平移。
- 平移范围基于当前缩放后的图片尺寸与可视区域边界进行裁剪，避免图片被完全拖出视口。
- 鼠标指针样式根据是否可拖动实时变化。

### 关闭规则

以下入口全部走同一关闭守卫：

- 键盘 `Esc`
- 右上角关闭按钮
- 浏览器返回

无未保存更改时直接关闭；有未保存更改时先弹出确认框，确认后才放弃草稿并离开。

## 右侧信息栏设计

右侧信息栏采用“专业信息卡片式分组 + 固定底部操作区”的结构，但页面本身不使用悬浮卡片嵌套。每个分组都使用稳定的小标题、标签列和值列，保持专业看图工具的阅读习惯。

### 信息分组

#### 文件信息

- 名称
- 格式
- 大小
- 像素
- 上传时间

#### 拍摄信息

- 拍摄时间
- 手动位置标签
- EXIF GPS 坐标
- 手动覆盖坐标

#### 相机与镜头

- 相机品牌
- 相机型号
- 镜头型号

#### 曝光参数

- 焦距
- 光圈
- 快门
- ISO

#### 更多元数据

- 默认折叠
- 以“字段名 / 值”方式整齐展示 `raw EXIF` 中尚未被结构化字段覆盖的内容
- 尽量保留专业摄影字段，不只展示少量摘要

### 缺失值策略

- 没有数据时不伪造内容。
- 结构化区块仍保留标题和空值样式，保证信息栏结构稳定，不因字段缺失而发生大幅跳动。
- 原始元数据区只展示实际可序列化且可读的字段，不展示对象类型的无意义嵌套内容。

## 标签编辑模型

标签编辑区需要满足专业管理界面的可读性与稳定性：

- 已有标签使用醒目的 pill 展示。
- 新增标签通过下拉面板选择。
- 下拉面板层级必须高于地图，避免被地图区域遮挡。
- 新增和删除标签都只更新本地草稿，不立即请求接口。
- 支持在保存前多次调整标签组合。

标签编辑的最终状态由统一保存按钮提交。

## 地图与位置编辑模型

### 草稿来源

位置编辑的初始值按以下优先级计算：

1. 手动覆盖位置 `image.location`
2. EXIF GPS 坐标
3. 无坐标时使用默认地图视野

### 交互方式

地图组件需要同时支持三种输入方式，并且它们始终同步到同一份草稿状态：

- 点击地图落点
- 拖动地图上的 pin marker
- 手动输入纬度和经度

当经纬度输入为合法数值时：

- 地图 marker 实时移动到新坐标
- 地图中心根据需要回到该坐标附近

### Marker 语义

- 地图上使用 pin 风格图标表示图片位置。
- 不使用图片缩略图作为 marker。
- marker 需要支持拖动。

### 位置文本信息

位置编辑区包括：

- 纬度输入框
- 经度输入框
- 地点标签输入框

地点标签允许为空。保存时若为空，则以 `null` 写入数据库。

### 清除手动位置

侧栏提供“清除手动位置”操作：

- 如果当前图片存在手动覆盖位置，则允许删除该覆盖。
- 删除后界面回退到 EXIF GPS 作为展示和编辑基础。
- 如果原图也没有 EXIF GPS，则地图回到默认视野，marker 清空。

## 统一草稿态与保存模型

右侧侧栏维护统一草稿状态：

- `draftTagIds`
- `draftLatitude`
- `draftLongitude`
- `draftLocationLabel`
- `isDirty`

### 脏数据判断

只要以下任一内容和初始值不同，即视为有未保存更改：

- 标签集合
- 纬度
- 经度
- 地点标签

### 保存行为

右侧底部提供统一的“保存更改”按钮。点击后一次性提交：

- 标签集合
- 位置经纬度
- 地点标签

保存成功后：

- 页面保持在当前详情页
- 侧栏使用服务端返回的新数据重置草稿
- `isDirty` 置为 `false`
- 地图与标签 UI 同步到新的持久化状态

### 重置草稿

右侧底部同时提供“重置草稿”操作：

- 放弃当前所有本地编辑
- 恢复到详情页初始加载时的标签和位置状态
- 清空错误提示

## 服务端接口设计

现有只读接口 `GET /api/images/[id]` 扩展为同时支持聚合更新 `PUT /api/images/[id]`。

### 请求体

请求体一次性携带：

- `tagIds: string[]`
- `location`
  - `latitude: number`
  - `longitude: number`
  - `label?: string`

当用户选择清除手动位置时，前端提交：

- 合法的 `tagIds`
- `location: null`

### 服务端行为

在单个数据库事务中完成以下动作：

1. 校验当前用户已登录。
2. 校验图片存在且未删除。
3. 校验所有 `tagIds` 都有效。
4. 删除旧的 `ImageTag` 关系并写入新的 `ImageTag` 集合。
5. 如果 `location` 为 `null`，删除该图片已有的 `ImageLocationOverride`。
6. 如果 `location` 为合法坐标，则对 `ImageLocationOverride` 执行 `upsert`。
7. 记录审计日志，注明本次属于统一编辑保存。

### 返回值

保存成功后返回：

- 最新标签列表
- 最新手动位置覆盖信息
- 如有需要，可返回整理后的图片详情摘要，供前端刷新本地状态

## 关闭确认

关闭确认框复用现有确认对话框组件能力，但文案改为适合详情页编辑场景：

- 标题：检测到未保存更改
- 描述：当前对标签或位置信息的修改尚未保存，确认离开将丢失这些更改
- 主按钮：放弃更改
- 次按钮：继续编辑

该确认框由详情页统一控制，而不是由标签区或地图区各自控制。

## 错误处理

### 前端

- 标签、位置校验失败时，在侧栏编辑区内显示明确错误，不关闭详情页。
- 经纬度格式非法时禁用保存按钮，或在保存前给出字段级错误。
- 保存请求失败时保留草稿，不丢弃用户已编辑内容。

### 服务端

- 未登录返回 `401`
- 图片不存在返回 `404`
- 标签非法或坐标超出范围返回 `400`
- 数据库事务失败返回通用错误，并保证标签和位置不会出现部分成功、部分失败的状态

## 测试策略

### 前端交互验证

需要验证以下核心交互：

- 从缩略图进入详情页时的放大动画可以正常执行。
- 滚轮缩放、按钮缩放和重置可以稳定工作。
- 缩放后图片可拖动，且不会被完全拖出视口。
- 有未保存更改时，`Esc`、关闭按钮和浏览器返回都会先弹确认框。
- 经纬度手动输入后，地图 marker 和地图中心实时回显。
- 点击地图或拖动 pin 后，经纬度输入框实时同步。
- 标签下拉面板不会被地图遮挡。

### 自动化测试

新增或修改的自动化测试应覆盖：

- `PUT /api/images/[id]` 统一保存标签和位置时的事务行为
- `location = null` 时删除手动覆盖位置
- `location` 为合法值时正确 `upsert` 手动覆盖位置
- 非法标签或非法坐标时返回正确错误
- 草稿状态在保存成功后重置，在保存失败时保留

## 本次范围外

以下内容不纳入本次重构范围：

- 将手动位置或标签改写回原始 OSS 图片 EXIF
- 在详情页中编辑图片标题、描述或其他主数据字段
- 多图切换、幻灯片播放或键盘上一张/下一张浏览
- 在分享页复用这一套完整编辑能力
