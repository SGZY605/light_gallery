# 分享照片选择设计

## 目标

调整分享页创建链接的逻辑，让用户在创建链接前明确选择要分享的照片。标签只用于筛选选择浮窗中的可见图片，不再作为分享链接内容的动态规则。

本次设计需要满足：

- 分享页提供一个浮窗，展示当前用户所有未删除图片的方形缩略图。
- 浮窗内可通过页面标签筛选可见图片。
- 每张缩略图左上角有复选框，勾选表示该图片会被分享。
- 浮窗提供“全选当前筛选结果”按钮，只选择当前筛选后可见的图片。
- 用户可以单独勾选或取消任意图片。
- 点击右侧“创建链接”后，按当前已选照片创建分享链接。
- 分享链接 token、过期、撤回、下载权限等逻辑保持现有设计。
- 当分享链接已经不可用，也就是已过期或已撤回时，允许手动删除该分享链接记录。

## 当前状态

现有分享模型是 `Share` 通过 `ShareTag` 关联标签，公开分享页调用 `getImagesForShare()` 按标签动态查询图片。

这会产生两个与新需求冲突的行为：

- 创建链接时没有明确的照片选择清单。
- 分享链接里的图片会随图片标签变化而变化，无法表达“创建时选中的照片”。

因此本次需要新增分享与图片的固定关联。

## 核心决策

### 1. 分享链接绑定照片快照

新增 `ShareImage` 关联表，将 `shareId` 与 `imageId` 绑定。创建分享时写入当前选中的图片 ID 列表。

公开分享页只读取 `ShareImage` 关联图片，不再按标签动态匹配图片。这样链接内容稳定，符合“当前选中的照片创建分享链接”的语义。

### 2. 标签只负责浮窗筛选

分享页仍加载用户标签，并在创建区域中作为筛选控件展示。标签筛选仅影响浮窗内哪些图片可见，不参与后端分享内容规则。

创建链接提交的是 `imageIds`，不是由 `tagIds` 推导的动态图片集合。

### 3. 保留分享标签展示

创建分享时可以继续保存当前筛选标签到 `ShareTag`，用于后台列表和公开页展示“这条分享来自哪些筛选标签”。这些标签不再决定公开页实际图片集合。

如果用户没有选择筛选标签，但手动选择了图片，也允许创建分享；公开页只展示图片。

### 4. 只允许删除不可用分享记录

分享记录状态继续由 `getShareState()` 判断：

- `active`：可复制、可撤回，不可删除记录。
- `expired`：链接不可用，可删除记录。
- `revoked`：链接不可用，可删除记录。

删除记录会删除 `Share` 本身，并通过级联删除关联的 `ShareTag` 和 `ShareImage`。

## 数据模型

新增模型：

- `ShareImage`
  - `shareId`
  - `imageId`
  - `createdAt`
  - 主键：`[shareId, imageId]`
  - 索引：`imageId`
  - `share` 关系使用 `onDelete: Cascade`
  - `image` 关系使用 `onDelete: Cascade`

`Share` 新增：

- `images: ShareImage[]`

`Image` 新增：

- `shares: ShareImage[]`

## 后端行为

### 创建分享

服务端动作与 `/api/shares` 都接受 `imageIds`。

校验规则：

- 用户必须已登录。
- 标题不能为空。
- `imageIds` 去重后至少 1 个。
- 所有图片必须属于当前用户，且 `deletedAt` 为 `null`。
- 如果提交了 `tagIds`，所有标签必须属于当前用户。
- `expiresAt`、`allowDownload` 维持现有语义。

创建记录时写入：

- `Share`
- `ShareImage[]`
- 可选 `ShareTag[]`

### 公开分享查询

`getImagesForShare()` 改为根据 `ShareImage` 查找图片：

- 图片必须 `deletedAt: null`
- 图片必须仍属于分享创建者
- 结果按 `ShareImage.createdAt` 或图片创建时间稳定排序
- 继续 include `exif` 和 `tags`

### 删除分享记录

新增服务端动作和 API 删除能力：

- 仅当前分享创建者可删除自己的分享记录。
- 只有 `expired` 或 `revoked` 状态可以删除。
- `active` 分享删除请求返回或静默拒绝。
- 删除后刷新 `/dashboard/shares`。

## 前端交互

分享页创建区域分为两块：

- 左侧：标题、描述、标签筛选、图片选择浮窗入口。
- 右侧：过期时间、下载权限、创建链接按钮、已选数量。

浮窗行为：

- 打开后展示所有当前用户未删除图片的方形缩略图。
- 点击标签筛选时只改变浮窗可见图片。
- 每张图左上角复选框控制该图是否入选。
- 点击“全选当前筛选结果”后，把当前可见图片全部加入已选集合。
- 用户仍可逐张取消或新增选择。
- 选择状态不因筛选变化丢失。
- 创建按钮提交当前已选图片。

## 错误处理

- 未选择图片时不创建分享。
- 图片或标签不属于当前用户时拒绝创建。
- 删除记录时如果分享仍可用，则不删除。
- 公开页遇到已删除记录沿用现有“不存在或已被删除”的提示。

## 测试策略

### 单元与集成测试

- `getImagesForShare()` 读取 `ShareImage` 关联，而不是按标签动态匹配。
- 创建分享 API 校验 `imageIds` 必须属于当前用户且未删除。
- 删除分享记录只允许 `expired` 或 `revoked`。
- 分享页源码契约包含图片选择组件、`imageIds` 提交、不可用链接删除动作。

### 前端选择逻辑测试

抽出小型纯函数 helper 测试：

- 标签筛选只影响可见图片。
- 全选只选择当前可见图片。
- 单图切换不会影响其他选择。
- 筛选变化不会丢失已选图片。
